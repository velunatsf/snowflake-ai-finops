Exercise 1

Sentiment Analysis

Lowest Cost AI_SENTIMENT()

Context: Analyze customer feedback sentiment using Snowflake's most cost-efficient AI function.

Approach A: New AI_SENTIMENT (returns OBJECT with category strings)

SQL 
SELECT
    customer_id,
    customer_name,
    LEFT(feedback_text, 80)                     AS feedback_preview,
    AI_SENTIMENT(feedback_text)                 AS sentiment_result,
    AI_SENTIMENT(
        feedback_text
    ):categories[0]:sentiment::VARCHAR          AS sentiment_label
FROM cortex_lab.ai_workshop.customer_feedback
LIMIT 20;

Approach B: Legacy SENTIMENT (returns numeric score -1 to 1)

SQL 
SELECT
    customer_id,
    customer_name,
    LEFT(feedback_text, 80)                     AS feedback_preview,
    SNOWFLAKE.CORTEX.SENTIMENT(
        feedback_text
    )                                           AS sentiment_score,
    CASE
        WHEN SNOWFLAKE.CORTEX.SENTIMENT(feedback_text) > 0.3
             THEN 'Positive'
        WHEN SNOWFLAKE.CORTEX.SENTIMENT(feedback_text) < -0.3
             THEN 'Negative'
        ELSE 'Neutral'
    END                                         AS sentiment_label
FROM cortex_lab.ai_workshop.customer_feedback
LIMIT 20;
AI_SENTIMENT vs SENTIMENT

AI_SENTIMENT() (new) returns an OBJECT: {"categories":[{"name":"overall","sentiment":"positive"}]}. Extract via :categories[0]:sentiment::VARCHAR.

SNOWFLAKE.CORTEX.SENTIMENT() (legacy) returns a float from -1.0 to 1.0. Both are highly cost-efficient - no model parameter required.

Exercise 2

Text Classification

Lower Cost AI_CLASSIFY()

Context: Route customer feedback to support teams by categorizing into predefined topics.

SQL 
SELECT
    customer_id,
    market_segment,
    LEFT(feedback_text, 80)                     AS feedback_preview,
    AI_CLASSIFY(
        feedback_text,
        ['billing',
         'technical_issue',
         'general_inquiry',
         'complaint',
         'praise']
    ):labels[0]::VARCHAR                        AS category
FROM cortex_lab.ai_workshop.customer_feedback
LIMIT 20;
AI_CLASSIFY vs AI_COMPLETE

AI_CLASSIFY() is significantly cheaper than using AI_COMPLETE() with a classification prompt. Always use the specialized function when available.

Exercise 3

Text Extraction

Medium Cost mistral-7b

Context: Extract structured information from unstructured text using a budget-tier model.

SQL 
SELECT
    customer_id,
    feedback_text,
    AI_COMPLETE(
        'mistral-7b',
        CONCAT(
            'Extract the main complaint topic from this customer ',
            'feedback in 3 words or less. ',
            'Reply with only the topic, nothing else. ',
            'Feedback: ', feedback_text
        )
    )                                           AS extracted_topic
FROM cortex_lab.ai_workshop.customer_feedback
WHERE LENGTH(feedback_text) > 50
LIMIT 10;
Model Choice Matters

Notice we use mistral-7b - the budget tier model. For a simple 3-word extraction task, a premium model would cost 10–20× more with no quality gain.

Exercise 6

Boolean Filtering with AI_FILTER

Lower Cost AI_FILTER()

Context: Filter rows using natural language conditions instead of writing complex LIKE/REGEXP patterns. AI_FILTER returns TRUE/FALSE for each row based on a text description.

SQL 
-- AI_FILTER: Find orders with urgent/escalation language
SELECT
    O_ORDERKEY,
    O_ORDERPRIORITY,
    O_COMMENT,
    SNOWFLAKE.CORTEX.AI_FILTER(CONCAT(
        O_COMMENT,
        'This text mentions a delay, complaint, or urgent issue'))
FROM cortex_lab.ai_workshop.ORDERS
LIMIT 10;
AI_FILTER Syntax

AI_FILTER takes a single argument: the text concatenated with a natural language description of the boolean condition. Use it as a cheaper alternative to AI_COMPLETE() for yes/no filtering tasks.

Common Pitfalls
  • Can use AI_FILTER directly (no prefix needed) or SNOWFLAKE.CORTEX.AI_FILTER (fully qualified)
  • The argument is CONCAT(column, 'description') - not two separate arguments
  • Returns a boolean, so use directly in WHERE or SELECT

Use in WHERE clause

SQL 
-- Filter only rows where the condition is TRUE
SELECT
    O_ORDERKEY,
    O_ORDERPRIORITY,
    O_COMMENT
FROM cortex_lab.ai_workshop.ORDERS
WHERE SNOWFLAKE.CORTEX.AI_FILTER(CONCAT(
    O_COMMENT,
    'This text mentions a delay, complaint, or urgent issue'))
LIMIT 10;

Key Takeaways

  1. AI_SENTIMENT() is the cheapest function - use it for sentiment, not AI_COMPLETE()
  2. AI_CLASSIFY() is cheaper than AI_COMPLETE() for classification tasks
  3. AI_FILTER() is cheaper than AI_COMPLETE() for boolean yes/no filtering
  4. Model choice can mean 10–50x cost difference for the same task
  5. Always test small, measure cost, then project before scaling
  6. The right model is the cheapest one that meets your quality requirements
