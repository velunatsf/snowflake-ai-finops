MODULE 04 5 min

CoCo Setup

Install the Snowflake CoCo CLI and connect to your Snowflake account

Learning Objectives

  • Install the Snowflake CoCo CLI on your workstation
  • Connect to your Snowflake account via Browser-Based SSO
  • Run your first prompt and verify the connection

Requirements

  • Snowflake account with cortex_analyst role configured (Module 03)
  • Terminal with bash, zsh, or fish (macOS/Linux) or PowerShell (Windows)
  • The SNOWFLAKE.CORTEX_USER database role (granted in Module 03)
A

CLI Installation

1

macOS and Linux (including WSL)

Open a terminal and run the installer:

Bash 
curl -LsS https://ai.snowflake.com/static/cc-scripts/install.sh | sh

The script installs the cortex executable to ~/.local/bin and adds it to your PATH.

2

Windows (Native - Preview)

Open PowerShell and run:

PowerShell 
irm https://ai.snowflake.com/static/cc-scripts/install.ps1 | iex

The executable installs to %LOCALAPPDATA%\cortex and is added to your PATH automatically.

Installation Output:

The installer downloads the latest version, verifies the checksum, and installs Cortex Code. The warning about symlinks on Windows is expected and does not affect core functionality.

3

Verify Installation

Confirm Snowflake CoCo CLI is installed correctly:

Bash 
cortex --version

You should see the installed version number printed to the terminal.

B

Connect to Cortex

1

Launch Snowflake CoCo CLI

Start the CLI by typing cortex in your terminal:

Bash 
cortex

You'll see the Cortex Code welcome screen:

2

Set Up Agent Account

On first launch, the CLI will prompt you to set up your agent account. Select "Sync from app.snowflake.com" to import your existing Snowflake connection:

Available options:

  • Sync from app.snowflake.com - Import connections from your Snowflake account (recommended)
  • Enter connection details manually - Type account identifier, username, and auth method
  • Sign up for a free Cortex Code trial - Create a new trial account
3

Choose Your Connection

If you have multiple Snowflake accounts, select the one you configured in Module 03:

The connection list shows your account name, account identifier, and associated user. Select the account where you created the cortex_analyst role.

4

Browser-Based SSO Authentication

When you select externalbrowser as the authenticator (the default), the following flow occurs:

What happens during Browser-based SSO:

  1. CLI opens your default browser to the Snowflake login page
  2. You authenticate with your Snowflake credentials (MFA required)
  3. Browser redirects back to a local callback URL
  4. CLI receives the authentication token and stores it for the session
connections.toml Configuration

After first connection, your config is saved in ~/.snowflake/connections.toml:

TOML 
[connections.my_account]
account = "ORGNAME-ACCTNAME"
user = "YOUR_USER"
authenticator = "externalbrowser"
role = "cortex_analyst"
warehouse = "cortex_wh"
database = "cortex_lab"
Troubleshooting SSO
  • If the browser does not open automatically, copy the URL from the terminal and paste it manually
  • Ensure pop-ups are not blocked for the Snowflake domain
  • If using a remote/headless environment, switch to Programmatic Access Token (PAT) instead
  • Token expires after the session ends. Re-running cortex will re-authenticate via browser
5

Verify Connection

Once connected, verify your session status:

Cortex Code 
/status

This prints your active connection, role, warehouse, database, and schema.

C

Run Your First Prompt

1

Ask a Plain-English Question

Type a natural language query:

Cortex Code 
What databases do I have access to?

Cortex Code translates your request into SQL, runs it against Snowflake, and returns the results directly in the terminal.

2

Try More Examples

Cortex Code 
Show me the top 10 largest tables in my account by row count
Cortex Code 
Explain what the SNOWFLAKE.ACCOUNT_USAGE.QUERY_HISTORY view contains
Build the Cost-Check Habit

Your first prompt cost tokens. At this point, you should already be thinking: "How much did that cost?" This cost awareness habit is what we're building throughout this training. In Module 06, you'll learn to query your actual token consumption.

D

Useful CLI Commands

Command Description
/status Show current connection, role, warehouse, database, schema
/help List all available slash commands
/sql SELECT ... Run SQL directly
/new Start a fresh session
/resume List and resume previous sessions
/rename <name> Rename the current session
/rewind Roll back to an earlier point in the conversation
/fork Create a new session branched from current point
Ctrl+T Open the built-in table viewer for query results
#TABLE_NAME Reference a table - pulls schema and sample rows into context
Table Reference Shortcut

Prefix a fully qualified table name with # to automatically pull its schema and sample rows into the conversation:

Cortex Code 
Tell me about #SNOWFLAKE.ACCOUNT_USAGE.WAREHOUSE_METERING_HISTORY
