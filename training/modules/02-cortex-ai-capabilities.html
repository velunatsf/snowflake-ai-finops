Function Selection Strategy

Start with specialized functions (AI_SENTIMENT, AI_CLASSIFY) before reaching for AI_COMPLETE(). Specialized functions are optimized for their specific task and typically cost less than using a general-purpose LLM for the same job.

Context Window = Capacity, Not Cost

The context window (e.g. 32K tokens) is the maximum a model can process per call - you only pay for tokens actually used. If your prompt is 500 tokens and the response is 200 tokens, you pay for 700 tokens, not 32K.

C

Function Deep Dive

AI_SENTIMENT(text [, categories])

Returns an OBJECT with sentiment categories (positive, negative, neutral, mixed, unknown)

Lowest Cost
Best For

Customer feedback, support tickets, reviews, social media

Cost

0.095 credits per 1M characters

Notes

No model selection - Snowflake-optimized. Optional categories array for aspect-level sentiment.

FinOps Tip

AI_SENTIMENT is the most cost-efficient Cortex function for sentiment analysis. Always use it instead of prompting AI_COMPLETE() for sentiment.

AI_COMPLETE(model, prompt)

General-purpose LLM generation with model selection

Variable Cost
Best For

Classification, extraction, generation, complex reasoning

Cost

Varies widely by model (see Model Cards above)

Notes

First parameter is model name as string

Cost Warning

Most expensive function category. Always specify the cheapest model that meets your quality requirements. The cost difference between mistral-7b and claude-4-sonnet can be 50× or more.

AI_SUMMARIZE_AGG(expr)

Aggregate function - summarizes a column of text data across multiple rows

Moderate Cost
Best For

Summarizing columns of reviews, call transcripts, logs, meeting notes

Cost

Uses underlying model rates (typically llama-based)

Notes

Handles datasets larger than max context window. For single-row summarization, use AI_COMPLETE with a summarize prompt.

AI_TRANSLATE(text, source_lang, target_lang)

Translates text between 24+ languages with auto-detect support

Moderate Cost
Best For

Multilingual content, localization, international data

Cost

0.333 credits per 1M characters

Languages

en, es, fr, de, it, pt, ja, ko, zh, and 80+ more

AI_CLASSIFY(text, categories)

Zero-shot classification into provided categories (single or multi-label)

Lower Cost
Best For

Ticket routing, content tagging, intent detection

Cost

Lower than equivalent AI_COMPLETE() prompt

Notes

Pass categories as an array. Access result via :labels[0]::VARCHAR

FinOps Tip

More cost-efficient than AI_COMPLETE() for classification tasks. Use it instead of prompting an LLM with "Classify this text as one of the following..."

AI_EXTRACT(text, fields)

Extracts structured fields from unstructured text

Moderate Cost

Provide text + array of field names to extract

D

Cortex Code

Cortex Code is Snowflake's AI-powered coding assistant - available in both Snowsight (web UI) and as a CLI for local development. It adds capabilities beyond the SQL functions covered above.

What Cortex Code Adds

  • Natural language → SQL generation
  • Code explanation and documentation
  • Agentic workflows and multi-step reasoning
  • CLI-driven interaction for developers
  • Integration with MCP servers and external tools

Cost Tracking for Cortex Code

  • CORTEX_CODE_SNOWSIGHT_USAGE_HISTORY
  • CORTEX_CODE_CLI_USAGE_HISTORY
  • Tracks user, timestamp, tokens, credits
  • CREDITS_GRANULAR JSON breakdown by model
Key Insight

Cortex Code is the AI layer that helps you write the SQL that calls the other Cortex functions. It accelerates development of your AI pipelines. In Module 07, we'll use Cortex Code to build the Streamlit dashboard that tracks all Cortex AI spend.

Knowledge Check

Q1: Which Cortex function is most cost-efficient for analyzing customer feedback sentiment?

AI_SENTIMENT() - It uses Snowflake's optimized internal model at 0.095 credits per 1M characters. Using AI_COMPLETE() with a prompt like "Rate the sentiment of this text" would cost significantly more.

Q2: When should you use AI_CLASSIFY() instead of AI_COMPLETE()?

Use AI_CLASSIFY() when you need to categorize text into a set of predefined categories. It's more cost-efficient and purpose-built for this task. Only use AI_COMPLETE() for classification when you need reasoning, explanations, or categories that emerge dynamically from the text.

Q3: What's the cost difference between mistral-7b and claude-4-sonnet for the same workload?

Approximately 10-50× difference. mistral-7b processes ~8.3M input tokens per credit, while claude-4-sonnet processes ~667K input tokens per credit (input) and only ~133K output tokens per credit. For output-heavy workloads, the difference can be even greater.
